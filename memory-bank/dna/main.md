# Базовые правила

## Аннотированные ссылки

Аннотированные ссылки используются в `memory-bank/README.md` и состоят из ссылки на файл и краткого пояснения к ней.

Правило оформления аннотированной ссылки: после пункта со ссылкой должны идти две строки с описанием в формате `**Что:** ...` и `**Читать, чтобы:** ...`.

Все ссылки в README.md должны вести на существующие файлы

## Проверка новых файлов

Для новых untracked-файлов проверяй результат через `git status --short` и чтение самого файла.

Не полагайся на `git diff -- path` как единственную проверку нового файла: обычный diff не показывает содержимое untracked-файлов.

## start_task

При выполнении `.prompts/start_task` перед созданием нового `brief.md` смотри существующий `brief.md` похожей задачи, если он есть.

Это помогает сохранить локальный формат `memory-bank/features/{task_number}/brief.md` без расширения требований сверх текста issue.
