# Brief

## Цель

Привести `memory-bank` проекта к виду `https://github.com/dapi/memory-bank.git`.

## Основные требования из issue

- Изменение относится к `memory-bank`.
- Целевой ориентир указан как репозиторий `https://github.com/dapi/memory-bank.git`.
- Задача сформулирована как приведение текущего `memory-bank` к виду указанного репозитория.

## Ограничения и допущения

- Единственный первичный источник требований для этого шага: GitHub issue `#14` в репозитории `atamanuk/mapper`.
- Issue не содержит детального описания требуемой структуры, набора файлов или правил миграции.
- Не требуется достигать полного соответствия с целевым примером `memory-bank`; необходимость каждой папки нужно обсуждать отдельно.
- Изменения папок выполняются в ручном режиме: по каждой папке нужно отдельно согласовать действие перед изменением структуры.
- В текущей структуре репозитория `memory-bank` уже существует и содержит `README.md`, `domain/README.md`, `domain/problem.md`, проектные документы, документы по тестированию, `dna/README.md`, governance-документы в `dna`, восстановленный локальный `dna/main.md` для последующего ручного переноса информации и директории задач в `features`.
- Для папки `domain` выполнен ручной шаг: создан `domain/README.md` по целевому примеру, `project/overview.md` перенесён в `domain/problem.md` с frontmatter, ссылки скорректированы.
- Demo-архитектура перенесена из `project/demo_architecture.md` в `domain/demo/architecture.md` с frontmatter и ссылкой из `domain/README.md`.
- На этом шаге фиксируется только краткий `brief`, без перехода к детальной спецификации или плану реализации.
