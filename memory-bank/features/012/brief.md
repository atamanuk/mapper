# Brief

## Цель

Добавить в demo-проект возможность по кнопке `Apply` применить к экземпляру целевой demo-модели `Json Patch`, полученный после мэппинга.

## Основные требования из issue

- Изменение относится к demo-проекту.
- После мэппинга `Json Patch` в demo должен быть доступен отдельный сценарий его применения к экземпляру целевой demo-модели.
- Применение должно запускаться отдельным пользовательским действием по кнопке `Apply`.
- После нажатия `Apply` результат применения должен быть наблюдаем в demo: пользователь должен увидеть состояние объекта после применения `Json Patch`.

## Проверяемый результат

- Пользователь может выполнить мэппинг исходного `Json Patch` в demo и затем отдельно запустить действие `Apply`.
- Действие `Apply` использует `Json Patch`, полученный после мэппинга, а не исходный документ.
- После выполнения `Apply` demo показывает состояние целевой модели после применения patch, чтобы результат можно было проверить визуально.

## Ограничения и допущения

- Единственный первичный источник требований для этого шага: GitHub issue `#12` в репозитории `atamanuk/mapper`.
- В issue не описаны точный вид UI и поведение при ошибках.
- По структуре репозитория demo-приложение находится в `src/Mapper.Demo`.
- На этом шаге фиксируется только краткий `brief`, без перехода к детальной спецификации или плану реализации.
